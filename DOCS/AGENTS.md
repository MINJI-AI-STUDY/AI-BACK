# 저장소 작업 규칙

## 저장소 기준
- 문서 구조는 `00_기준`과 `15_기능명세`를 분리합니다.
- 구현 중 생성되는 임시 계획/메모/태스크는 `.sisyphus/` 또는 `.omx/`에 둡니다.
- current 문서가 아닌 참고 문서는 필요 시에만 읽습니다.

## 작업 순서
1. `DOCS/README.md`에서 current 범위를 확인합니다.
2. 공통 규칙은 `DOCS/00_기준/`에서 확인합니다.
3. 기능별 작업은 `DOCS/15_기능명세/`에서 범위를 좁힙니다.
4. 실행 자산은 `.sisyphus/`에 남기고, 재사용 검증 후 기준 문서로 승격합니다.

## 코드/문서 원칙
- 공통 규칙과 기능 요구사항을 혼합하지 않습니다.
- 기능 구현 전에 PRD → USM → 기능명세 순으로 범위를 잠급니다.
- 문서 파일명은 역할이 드러나게 유지합니다.

## 검증 원칙
- 완료 처리 전 변경 파일 구조와 링크가 유효한지 확인합니다.
- 새 규칙이 생기면 즉시 개인 메모가 아니라 `.sisyphus/notepads/`에 남깁니다.
