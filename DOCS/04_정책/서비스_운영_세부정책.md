# 서비스 운영 세부정책

## 목적
- MVP 구현/운영/배포에 필요한 실제 서비스 정책을 세부 수준까지 고정합니다.

## 공통 입력 정책
- 모든 문자열 입력은 앞뒤 공백을 제거하고 저장합니다.
- 빈 문자열은 유효 입력으로 허용하지 않습니다.
- 서버 검증 실패 시 `code`, `message`, `retryable` 형식으로 응답합니다.

## 인증 및 계정 정책
- MVP 계정 역할은 `TEACHER`, `STUDENT`, `OPERATOR` 세 가지로 고정합니다.
- 데모 계정은 seed 데이터로 제공합니다.
- access token은 프론트 `sessionStorage`에만 저장합니다.
- 비인가 접근은 즉시 차단하고 역할 불일치 시 접근 불가 메시지를 표시합니다.

## 수업자료 업로드 정책
- 허용 파일 형식은 PDF 1종입니다.
- 업로드 파일 최대 크기는 `20MB`입니다.
- 자료 제목 최대 길이는 `100자`입니다.
- 자료 설명 최대 길이는 `300자`입니다.
- 저장 경로는 Railway persistent volume 기준 `/data/materials`입니다.
- 상태값은 `UPLOADED`, `PROCESSING`, `READY`, `FAILED` 4종만 사용합니다.
- `FAILED` 상태만 재처리 API 호출이 가능합니다.

## 문제 생성/검토/배포 정책
- 한 번의 생성 요청에서 문항 수는 `1~10개`만 허용합니다.
- 난이도는 `EASY`, `MEDIUM`, `HARD`만 허용합니다.
- 문항은 4지선다 객관식, 정답 1개 형식으로 고정합니다.
- 각 문항은 stem, options 4개, correctOptionIndex, explanation, conceptTags 1~2개를 반드시 가집니다.
- 교사는 stem/options/explanation/conceptTags/excluded를 수정할 수 있습니다.
- 제외되지 않은 문항이 1개 이상일 때만 배포할 수 있습니다.

## 학생 풀이/제출 정책
- 학생은 배포 코드로만 문제 세트에 입장합니다.
- 학생은 세트별 최종 제출 1회만 가능합니다.
- 임시 저장은 MVP 범위에서 제외합니다.
- 제출 시 모든 문항에 응답해야 합니다.
- 배포 세트에 `dueAt`이 있으면 마감 이후 제출을 차단합니다.

## 결과/대시보드 정책
- 학생 결과는 총점, 문항별 정오답, 해설을 제공합니다.
- 교사 대시보드는 학생별 점수, 문항별 정답률, 취약개념 태그를 제공합니다.
- 운영자 대시보드는 평균 점수, 참여율, 완료율만 제공합니다.
- 취약개념은 오답 문항 concept tag count 내림차순으로 계산합니다.

## 자료 기반 AI 질의응답 정책
- 질문 최대 길이는 `500자`입니다.
- 빈 질문은 허용하지 않습니다.
- 답변에는 최대 `2개`의 근거 snippet을 노출합니다.
- 근거 부족 시 명확한 안내 메시지를 반환합니다.
- AI 연동 실패 시 현재 자료 기반 fallback 답변으로 응답을 닫습니다.

## 환경변수 및 배포 정책
- `.env`, `.env.local`, 인증서, 개인 토큰 파일은 저장소에 커밋하지 않습니다.
- 저장소에는 `.env.example`만 둡니다.
- Frontend는 `5173`, API는 `8080`, AI는 `8000` 포트를 기본으로 사용합니다.

## 운영상 추후 고도화 항목
- 실제 PDF 파싱 품질 정책
- LLM/RAG 기반 근거 검증 정책
- 토큰/세션 보안 강화 정책
- SQLite에서 운영 DB로의 이행 정책
