# ADR: 학교 가입/승인 모델

## 상태
- 승인됨

## 결정
- 학교 마스터는 공공 데이터 seed를 내부 DB에 적재한다.
- 교직원 가입은 학교 이메일 도메인 검증 또는 학교 운영자 수동 승인으로 허용한다.
- 학생 가입은 학교 선택 + 실명 입력 + 학교 운영자 승인 후 활성화한다.
- 전체 시스템 운영자와 별도로 학교별 운영자 권한을 둔다.

## 이유
- MVP에서 외부 SSO/행정 시스템 연동 없이도 운영 가능한 최소 구조다.
- 학교 단위 tenant와 승인 권한을 일치시킬 수 있다.
- 법적 고지/동의 및 승인 로그 보관 요구를 추적하기 쉽다.

## 결과
- auth/admin 모델에 가입 요청과 승인 로그 테이블이 추가된다.
- 프론트는 로그인 외에 교직원/학생 가입 화면과 학교 운영자 승인 화면이 필요하다.
