# 인프라 운영 전략

## 현재 전제
- Front: Vite + React + TypeScript
- API: Spring Boot
- AI: FastAPI + LangChain + Gemini 2.5
- DB: SQLite
- Deploy: Vercel, Railway

## 운영 원칙
- 기술 구조 정보는 PRD와 기준 문서에 중복 없이 유지합니다.
- 인프라 확장 규칙은 확정 시 본 문서에 승격합니다.
- 업로드 PDF는 Railway API 서버의 persistent volume(`/data/materials`)를 기본 저장소로 사용합니다.
- 프런트(Vercel)는 업로드 파일을 직접 저장하지 않고 API 서버를 통해서만 접근합니다.

## 로컬 실행 원칙
- Frontend 로컬 포트는 `5173`을 사용합니다.
- API 로컬 포트는 `8080`을 사용합니다.
- AI 로컬 포트는 `8000`을 사용합니다.
- 로컬에서도 프론트는 API/AI 주소를 환경변수로 참조합니다.

## 환경변수 원칙
- 환경변수는 저장소에 직접 커밋하지 않습니다.
- 저장소에는 `.env.example`만 제공합니다.
- 실제 값은 로컬 또는 배포 환경에서만 주입합니다.
- 민감정보는 PR, 커밋, 리뷰 코멘트 어디에도 남기지 않습니다.
