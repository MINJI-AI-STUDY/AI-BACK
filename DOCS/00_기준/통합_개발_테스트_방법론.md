# 통합 개발 테스트 방법론

## 원칙
- PRD → USM → 기능명세 순으로 범위를 고정합니다.
- 구현 전에 수용 기준을 먼저 확인합니다.
- 테스트/검증 계획은 `.sisyphus/plans/` 또는 `.sisyphus/tasks/`에 연결합니다.

## 구현 진행 순서
- 명령을 받은 뒤 바로 구현하지 않고 세분화와 계획부터 수행합니다.
- 계획 구현 후 Oracle 리뷰를 거칩니다.
- 리뷰 결과를 반영한 뒤 커밋/PR/리뷰/재수정/merge 순으로 진행합니다.

## 코드 주석 원칙
- 프론트와 백엔드 모두 메서드 단위로 한글 주석을 작성합니다.
- 주석은 "왜 이 메서드가 필요한지"와 "무엇을 보장하는지"를 설명해야 합니다.
- 단순한 코드 번역 주석은 피하고, 도메인 규칙과 제약을 남깁니다.

## 검증 원칙
- 로컬 포트 기준으로 실제 접근 가능한 상태를 확인합니다.
- 빌드/테스트/헬스체크 결과를 근거로 완료를 판단합니다.

## 완료 근거
- 작업 산출물 위치가 명확해야 합니다.
- 수용 기준과 산출물이 연결되어야 합니다.
