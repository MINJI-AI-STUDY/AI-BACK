## Focus Scope

- 인증 성공/실패
- 역할 판별
- 역할별 라우팅
- 권한 없는 접근 차단

## Must Decide Before Coding
- Bearer JWT 사용
- 데모 계정은 SQLite seed 데이터로 제공
- 로그인 화면과 역할별 보호 라우트에서 가드 적용

## Done Criteria
- 교사/학생/운영자별 진입 경로가 구분된다.
- 권한 없는 화면 접근이 차단된다.
- 기능명세 수용 기준 4개가 모두 검증 가능 상태다.

## Blocked Conditions
- 없음. 사용자 role은 `TEACHER|STUDENT|OPERATOR`, token 저장은 `sessionStorage`로 잠금
