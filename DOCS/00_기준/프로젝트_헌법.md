# 프로젝트 헌법

## 목적
- 본 저장소는 문서 중심으로 제품 범위와 작업 규칙을 통제합니다.

## 우선순위
1. 공통 기준 준수
2. PRD/USM/기능명세 범위 준수
3. 구현 자산의 재사용 가능성 확보

## 문서 분리 원칙
- 공통 기준은 `00_기준`
- 기능 요구는 `15_기능명세`
- 실행 자산은 `.sisyphus`, `.omx`

## 저장소 운영 원칙
- 프론트/백엔드 저장소가 비어 있더라도 current 문서 기준을 먼저 잠그고 구현합니다.
- 구현과 리뷰와 머지는 항상 `main` 중심 GitHub Flow로 운영합니다.
- `AGENTS.md` 외 루트 markdown 산출물은 허용하지 않고, 모두 `DOCS/` 하위 적절한 위치에 둡니다.
