# 학교-학급 멀티테넌시 구현 계획

## 목적
- 학교 단위 논리 분리와 학급 단위 하위 구분을 백엔드/프론트 책임으로 나눠 구현합니다.

## 백엔드 할 일
1. 인증 사용자 모델에 `schoolId`, `classroomId` 추가
2. 자료/문제세트/제출/QALog에 `schoolId` 필수화
3. 학급 종속 기능에 `classroomId` 추가
4. 서비스 계층에서 school boundary 검증
5. 대시보드 집계와 결과 조회에 tenant filter 적용
6. 학교 간 데이터 혼선이 없도록 NotFound/Forbidden 기준 재정의

## 프론트엔드 할 일
1. 사용자 컨텍스트에 `schoolId`, `classroomId` 읽기 전용 표시
2. 교사/학생 workspace 상단 메타에 학교/학급 정보 노출
3. 학교/학급별 자료 목록/문제 목록 UX 정리
4. school/class 선택형 UI가 필요해지면 운영자 전용으로 제한 검토

## 단계 제안
- Phase 1: 인증/도메인 모델에 schoolId/classroomId 추가
- Phase 2: F2~F6 서비스 로직 tenant filter 적용
- Phase 3: 프론트 메타 표시 및 운영자 시야 정리

## 비목표
- 학교별 물리 DB 분리
- 학교별 배포 인프라 완전 분리
- 동적 tenant switching 지원
