# backend 구현 준비 허브

## 목적
- 기능 구현 전에 서버/API/AI/데이터 관점의 준비 범위를 고정합니다.

## 기능 분해

### F1 로그인 및 역할 기반 진입
- 인증/권한 경계 정의
- 사용자 역할 모델 정의
- 역할 기반 접근 정책 정리

### F2 수업자료 업로드 및 분석
- PDF 업로드 API 범위 정의
- 분석 상태 모델 정의
- 텍스트 추출/문서 분할/실패 상태 처리 기준 정리

### F3 객관식 문제 생성 및 배포
- 문제 생성 요청/응답 계약 초안
- 문제 세트/문항/배포 코드 데이터 구조 정리
- 교사 검토 후 배포 플로우 서버 기준 정리

### F4 학생 문제 풀이 및 자동 채점
- 문제 세트 조회/응답 저장/제출/채점 흐름 정리
- 중복 제출 정책 및 마감 상태 규칙 정리

### F5 결과 확인 및 대시보드
- 학생/교사/운영자 조회 권한 범위 정리
- 점수/정답률/취약개념/참여율 집계 기준 정리

### F6 자료 기반 AI 질의응답
- 자료 컨텍스트 검색 범위 정리
- 근거 부족 응답 규칙 정리
- 질문/답변 로그 저장 기준 정리

## 구현 시작 전 확인
- 각 기능별 API 계약 초안 존재 (`API_초안.md`)
- 주요 엔티티/상태 전이 정의 (`엔티티_및_상태전이.md`)
- 예외/권한/상태값이 기능명세 수용 기준과 연결됨
