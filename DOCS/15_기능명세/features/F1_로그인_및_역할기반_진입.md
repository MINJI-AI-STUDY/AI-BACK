# F1 로그인 및 역할 기반 진입

## 기능 요약
- 사용자는 로그인 후 자신의 역할에 맞는 홈 화면으로 진입합니다.

## API 명세

### POST `/api/auth/login`
- 목적: 로그인 수행
- 요청: `loginId`, `password` (교직원/운영자)
- 응답: `accessToken`, `role`, `displayName`
- 권한: Public

### POST `/api/auth/student/login`
- 목적: 학생 PIN 로그인 수행
- 요청: `schoolId`, `studentCode`, `pin`
- 응답: `accessToken`, `refreshToken`, `role`, `displayName`, `schoolId`, `classroomId`, `studentCode`
- 권한: Public

### GET `/api/auth/me`
- 목적: 현재 로그인 사용자 조회
- 응답: `userId`, `schoolId`, `classroomId`, `role`, `displayName`, `studentCode`, `active`, `createdAt`, `privacyConsents`
- 권한: Authenticated

### POST `/api/auth/refresh`
- 목적: refresh token으로 access token 재발급
- 요청: `refreshToken`
- 응답: `accessToken`, `refreshToken`, `role`, `displayName`

### POST `/api/auth/logout`
- 목적: refresh token revoke 및 로그아웃 처리
- 요청: `refreshToken`

## 정책

### POL-AUTH-001 역할 기반 접근 정책
- 코드: `POL-AUTH-001`
- 내용: `TEACHER`, `STUDENT`, `OPERATOR` 외 역할은 허용하지 않습니다.

### POL-AUTH-004 학생 인증 정책
- 코드: `POL-AUTH-004`
- 내용: 학생은 학교, 학생 코드(studentCode), PIN 조합으로 별도 로그인 경로를 사용하며 교직원/운영자와 비밀번호 정책을 공유하지 않습니다.
- 학생 코드는 학교 범위 내에서 고유하며, 동일 실명의 학생이 허용됩니다.

### POL-AUTH-002 인증 토큰 저장 정책
- 코드: `POL-AUTH-002`
- 내용: 프런트엔드는 access token을 `sessionStorage`에 저장합니다.

### POL-AUTH-003 라우트 가드 정책
- 코드: `POL-AUTH-003`
- 내용: 비인증 또는 역할 불일치 사용자는 보호 라우트 접근이 차단됩니다.
