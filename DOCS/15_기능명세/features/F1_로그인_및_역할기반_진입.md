# F1 로그인 및 역할 기반 진입

## 기능 요약
- 사용자는 로그인 후 자신의 역할에 맞는 홈 화면으로 진입합니다.

## API 명세

### POST `/api/auth/login`
- 목적: 로그인 수행
- 요청: `loginId`, `password`
- 응답: `accessToken`, `role`, `displayName`
- 권한: Public

### GET `/api/auth/me`
- 목적: 현재 로그인 사용자 조회
- 응답: `userId`, `role`, `displayName`
- 권한: Authenticated

## 정책

### POL-AUTH-001 역할 기반 접근 정책
- 코드: `POL-AUTH-001`
- 내용: `TEACHER`, `STUDENT`, `OPERATOR` 외 역할은 허용하지 않습니다.

### POL-AUTH-002 인증 토큰 저장 정책
- 코드: `POL-AUTH-002`
- 내용: 프런트엔드는 access token을 `sessionStorage`에 저장합니다.

### POL-AUTH-003 라우트 가드 정책
- 코드: `POL-AUTH-003`
- 내용: 비인증 또는 역할 불일치 사용자는 보호 라우트 접근이 차단됩니다.
