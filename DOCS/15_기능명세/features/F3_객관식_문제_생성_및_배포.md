# F3 객관식 문제 생성 및 배포

## API 명세
- `POST /api/teacher/materials/{materialId}/question-sets/generate`
- `POST /api/teacher/channels/{channelId}/question-sets/generate`
- `GET /api/teacher/materials/{materialId}/question-sets`
- `GET /api/teacher/channels/{channelId}/question-sets`
- `GET /api/teacher/question-sets/{questionSetId}`
- `PATCH /api/teacher/question-sets/{questionSetId}/questions/{questionId}`
- `POST /api/teacher/question-sets/{questionSetId}/publish`
- `GET /api/student/channels/{channelId}/active-question-set`

### 배포 요청 필드
- `dueAt` (선택): 문제 세트 마감 일시

## 정책

### POL-QSET-001 생성 문항 수 정책
- 코드: `POL-QSET-001`
- 내용: 한 번의 생성 요청에서 문항 수는 `1~10개`만 허용합니다.

### POL-QSET-001-A 생성 요청 필드 정책
- 코드: `POL-QSET-001-A`
- 내용: 생성 요청은 `questionCount`, `difficulty`를 사용하며, 채널 경로 생성 시 `materialId`가 필수입니다.

### POL-QSET-001-B 채널 귀속 정책
- 코드: `POL-QSET-001-B`
- 내용: 문제세트는 `question_sets.channel_id`로 채널 귀속을 가지며, 채널 워크스페이스는 채널 기준 문제세트를 우선 조회합니다.

### POL-QSET-002 객관식 형식 정책
- 코드: `POL-QSET-002`
- 내용: 모든 문항은 4지선다, 정답 1개 형식으로 고정합니다.

### POL-QSET-003 해설/태그 필수 정책
- 코드: `POL-QSET-003`
- 내용: 각 문항은 explanation과 conceptTags 1~2개를 반드시 포함합니다.

### POL-QSET-004 검토 수정 정책
- 코드: `POL-QSET-004`
- 내용: 교사는 stem, options, correctOptionIndex, explanation, conceptTags, excluded만 수정할 수 있습니다.

### POL-QSET-005 배포 가능 조건 정책
- 코드: `POL-QSET-005`
- 내용: 제외되지 않은 문항이 1개 이상일 때만 배포할 수 있습니다.

### POL-QSET-006 배포 코드 정책
- 코드: `POL-QSET-006`
- 내용: 배포 코드는 문제 세트 ID의 뒤 6자리를 대문자로 사용합니다.

### POL-QSET-007 예외 처리 정책
- 코드: `POL-QSET-007`
- 내용: 분석 완료(`READY`) 자료만 문제를 생성할 수 있으며, 생성 결과가 비어 있으면 배포를 차단합니다.

### POL-QSET-008 마감 일시 정책
- 코드: `POL-QSET-008`
- 내용: teacher는 문제 세트 배포 시 `dueAt`를 함께 설정할 수 있고, student 제출 화면은 이를 마감 기준으로 사용합니다.
