# 학교 가입/승인 웨이브 계획

## 범위
- 전국 초중고 학교 마스터 데이터를 DB seed로 보유한다.
- 교직원은 학교 이메일 도메인 또는 학교 운영자 승인으로 검증한다.
- 학생은 학교 선택 + 실명 기반으로 가입하고 학교 운영자 승인을 거쳐 활성화한다.
- 학교별 운영자는 같은 학교 교사/학생 가입 요청을 승인/반려할 수 있다.
- 서비스 이용약관, 개인정보 수집·이용 동의, 학생 서비스 고지, 승인 로그 보관 정책을 문서와 화면에 반영한다.

## 구현 순서
1. 학교 마스터/가입 요청/승인 로그 데이터 모델 추가
2. 교직원/학생 가입 API 추가
3. 학교 운영자 승인/반려 API 추가
4. 프론트 가입/승인/동의 화면 추가
5. 문서/법적 고지/운영 정책 반영

## 이번 웨이브 원칙
- backend-first로 진행한다.
- school > channel 구조는 유지한다.
- 교직원 검증은 MVP 기준으로 학교 이메일 도메인 매칭 또는 운영자 승인으로 처리한다.
