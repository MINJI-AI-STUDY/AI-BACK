# 운영 환경변수 및 시크릿 가이드

## 필수 환경변수
- `DB_URL`
- `DB_USERNAME`
- `DB_PASSWORD`
- `APP_AI_BASE_URL`
- `APP_SCHOOL_API_BASE_URL`
- `APP_SCHOOL_API_KEY`
- `APP_SCHOOL_API_ENDPOINT`
- `APP_SCHOOL_API_PAGE_SIZE`
- `APP_AUTH_JWT_SECRET`
- `APP_AUTH_JWT_EXPIRATION_MS`

## 기본 원칙
- 운영 환경에서는 `application.properties` 기본값에 의존하지 않는다.
- JWT secret은 32바이트 이상 랜덤 문자열로 관리한다.
- DB 계정은 애플리케이션 전용 계정을 사용한다.
- `.env.example`은 예시만 제공하며 실제 비밀값은 저장소에 커밋하지 않는다.

## 권장 값
- `APP_AUTH_JWT_EXPIRATION_MS=86400000` (24시간)
- 운영 보안 정책상 더 짧은 만료가 필요하면 1~12시간 범위에서 조정

## 배포 체크리스트
1. PostgreSQL 접속정보 주입 확인
2. JWT secret 별도 secret manager 또는 CI/CD secret store 등록
3. 운영 DB와 테스트/개발 DB 분리 확인
4. seed 계정 제거 또는 초기 관리자 계정 재설정 여부 확인
5. 로그에 token/password 출력이 없는지 확인

## 현재 구현 기준 참고
- 비밀번호는 BCrypt 해시로 저장된다.
- Access token은 JWT(HMAC)로 서명된다.
- 로컬 human-test는 `application-local.properties` + H2 profile로 재현 가능하다.
- 전국 학교 마스터는 Open API 연동 경로(`POST /api/operator/schools/sync-master`)를 통해 적재할 수 있다.
- 운영자용 조회 API 초안
  - `GET /api/operator/schools`
  - `GET /api/operator/schools/{schoolId}/classrooms`
  - `GET /api/operator/users?schoolId=`

## 학교 마스터 Open API 동기화
- 권장 소스: 한국교육학술정보원(NEIS) 학교 기본정보 Open API 또는 공공데이터포털 학교정보 파일데이터
- MVP 구현은 `open.neis.go.kr/hub` 기반 동기화 경로를 제공합니다.
- 운영자는 `APP_SCHOOL_API_KEY`를 설정한 뒤 `POST /api/operator/schools/sync-master`로 학교 마스터를 동기화할 수 있습니다.
