# 우선순위 재평가 및 구현 계획

## 목적
- 기존에 추출한 높은~중간 우선순위 후보를 현재 MVP 범위 기준으로 다시 평가한다.
- backend-first, frontend-second 원칙으로 실제 구현할 범위만 잠근다.

## 결론

### 이번에 구현
1. teacher publish 시 due date 입력 UX
2. teacher home에서 첫 채널 생성 bootstrap CTA

### 이번에 보류
1. student 제출/질문 이력 목록
2. student 채널/클래스 배정 UX
3. teacher roster 강화
4. 채널 메시지 UX 강화
5. 챗봇 이미지 첨부 실제 처리
6. operator 검색/필터/페이지네이션

## 근거
- due date는 backend/API/정책이 이미 존재해 프론트만 보강하면 기능 완결성이 올라간다.
- 첫 채널 생성 CTA는 teacher onboarding dead-end를 제거하는 저비용 고효율 수정이다.
- 나머지는 데이터 모델 확장 또는 추가 API 설계가 필요해 다음 웨이브로 미룬다.

## 구현 순서
1. backend 문서/API/테스트 보강
2. frontend due date UX 추가
3. teacher home bootstrap CTA 추가
4. 테스트/브라우저 QA
5. PR 브랜치 반영
